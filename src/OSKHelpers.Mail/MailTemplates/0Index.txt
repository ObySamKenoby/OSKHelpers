------

Sample file to add to the template folder for sending emails.
Modify the list of templates as needed.
Credentials.txt is an example of a template used in production for sending credentials
automatically generated for users.

------

The folder contains the default templates for sending emails.
Files must be copied inside MailTemplates.

All the following files must be present:

Credentials.txt          Template for sending credentials to the user
OrdersReport.txt         Template for sending the order report
....

All files share the following characteristics:

- lines starting with the # character are considered comments;
- the file must contain a line starting with @Subject: containing the subject of the email to be sent.
  The subject may contain TAGs specific to each template.
  The subject must be contained within a single line;
- everything else in the file is considered the body of the email. You can use any HTML tag
  and the specific TAGs for each file to format the email itself;
- each template uses specific tags identified as {TAG}
  It is important to note that tags are case sensitive, so `TAG` is different from `Tag`.
  The tags available for each template are specified within the template itself.
- each template can have conditional sections that will be included only in particular cases,
  identified with <<COND>> .... <</COND>>
  As with tags, each condition will be specified within the template file.

A valid template file may have the following format:

----------------


# This is a comment
# {USERNAME} is a test tag...
@Subject: Test email for user {USERNAME}

Good morning <strong>{USERNAME}</strong><br/>
Welcome.<br/>
# The following section will be included only if the configuration does not provide for a single company
<<NOSINGLE>>The application is not configured for a single company.<br/><</NOSINGLE>>


----------------
